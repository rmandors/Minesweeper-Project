# 💣 Minesweeper on MARIE.js

A fully playable implementation of **Minesweeper** running on the **MARIE.js** simulator, rendered on its 16×16 graphical display. The board is generated externally via a Python script and loaded into the simulator's memory — all game logic runs in MARIE assembly.

## How it works

MARIE is a minimal architecture with no mouse support, no interactive keyboard, and no random number generation. To work around this, the project is split into two clear layers:

- **Python** handles everything data-related: board generation, mine placement, neighbor calculation, and memory block export.
- **MARIE** handles everything runtime-related: reading input, game state management, win/loss detection, and rendering to the display.

### Workflow

1. Open the [Colab notebook](https://colab.research.google.com/drive/1nu0tEDZgRo7gR0Hrk88ic7IQeCxWXEKR#scrollTo=1SlU4Ecyp8LA) and run it to generate your board.
2. Python randomly places mines, computes neighbor counts, and outputs `play_minesweeper.mas` — a complete MARIE assembly file with the board already embedded inside it.
3. Python also prints a full board visualization for reference.
4. Copy the entire contents of `play_minesweeper.mas` and paste it into [marie.js.org](https://marie.js.org).
5. Run the program — the game is live.

## Controls

Since MARIE reads input sequentially, each move is entered as three consecutive values via `INPUT`:

| Prompt | Value |
|--------|-------|
| Row | `1`–`16` |
| Column | `1`–`16` |
| Action | `1` = reveal · `2` = toggle flag |

**Example** — reveal the cell at row 6, column 10:
```
6
10
1
```

**Example** — place a flag at row 6, column 10:
```
6
10
2
```

---

## Board representation

The board is **16 × 16 = 256 cells**, stored in two separate memory structures:

### Real board *(generated by Python)*

| Value | Meaning |
|-------|---------|
| `-1` | mine |
| `0`–`8` | adjacent mine count |

### Visible board *(managed by MARIE)*

| State | Display color |
|-------|--------------|
| Hidden cell | Dark gray |
| Flagged cell | Yellow |
| Revealed mine | Black |
| Empty cell (value 0) | Light gray |
| Values 1–8 | Unique color per value |

---

## Python script

The script takes the number of mines as input, randomly places them on the 16×16 grid, computes neighbor counts for every cell, and outputs two things:

1. **`play_minesweeper.mas`** — the complete MARIE assembly file with your board embedded, ready to paste directly into [marie.js.org](https://marie.js.org).
2. **Board visualization** — shows mines and values 0–8 to verify correctness.

---

## MARIE implementation

The assembly program covers:

- Sequential `INPUT` reading for row, column, and action
- Coordinate validation
- Indexed memory access to both board structures
- Visible board updates
- Loss detection (mine revealed)
- Win detection (all safe cells revealed)
- Color-based rendering on the 16×16 display

---

## Game rules

- Revealing a mine ends the game immediately (**loss**).
- Each safe cell displays the count of mines in its 8 neighboring cells (0–8).
- Flags mark suspected mines and prevent accidental reveals.
- The game is won when every safe cell has been revealed.
